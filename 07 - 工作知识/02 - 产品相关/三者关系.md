# 3.1 产品逻辑（真实版）

### 一句话总览

> **句子现在的核心体系是「秒懂 + 秒回」，而「秒红」是后来为了抖音和小红书的公域获客场景，从秒回中单独拆出来的新产品线。**

---

## 一、产品的真实发展顺序

不是一开始就有三套产品，而是这样长出来的：

### 第一步：只有「秒回」（RPA 时代）

最早公司只有 **句子秒回**：

- 核心是：  
    **企业微信 + RPA + 关键词自动回复**
    
- 解决的问题：  
    客服接待、销售回复、简单自动化
    
- 典型场景：  
    教培、销售私域、企微社群
    

当时本质是：

> **“用 RPA 帮人回消息”**

---

### 第二步：有了 AI，于是做了「秒懂」（大脑）

后来 AI 出现，公司发现：

> 不能再靠关键词和流程写死逻辑了，要有「会思考的 AI」

于是内部先做了一个平台：**句子秒懂**

它的角色是：

- 用来搭智能体
    
- 定义角色、人设、流程、记忆
    
- 调优 AI 表现
    

然后再把它 **接入秒回**

于是变成：

> **秒回 = 聊天工作台 + 秒懂里搭的 AI**

也就是现在你看到的结构：  
**秒回只是“前台”，秒懂才是“大脑”**

---

### 第三步：抖音 / 小红书搞不了企微 RPA → 做了「秒红」

问题来了：

秒回是为 **企业微信私域**设计的  
而抖音、小红书是 **公域平台**

- 没法跑企微 RPA
    
- 只能用官方 / 非官方 API
    
- 但获客场景非常重要
    

于是公司做了一个决策：

> 把「秒回里用于获客的那一部分能力」，抽出来，  
> 单独做一个 **抖音 / 小红书专用版本** → 秒红

所以：

> **秒红 ≈ 抖音 / 小红书版的秒回**

它走的是：

- API 接口
    
- 平台消息流
    
- 公域获客
    

而不是：

- 企微 RPA
    
- 私域工作台
    

---

## 二、现在三者的真实分工

你给的那个「引流手 / 转化手 / 大脑」的说法是对的，但需要加一个历史注解：

---

### 句子秒懂（大脑）

**所有 AI 的源头**

- 定义：
    
    - 角色
        
    - 流程
        
    - 记忆
        
    - 意图识别
        
- 交付团队用它来搭智能体
    
- 秒回、秒红的 AI 都来自这里
    

👉 **它是 AI 能力工厂**

---

### 句子秒回（私域转化）

**原始主产品**

- 核心：
    
    - 企业微信
        
    - 聚合聊天
        
    - RPA
        
    - 成交、转化、运营
        
- 绑定：
    
    - 客服
        
    - 销售
        
    - 教培私域
        

👉 **它是赚钱的主战场**

---

### 句子秒红（公域获客）

**从秒回中拆出来的新场景**

- 专为：
    
    - 抖音
        
    - 小红书
        
- 用来：
    
    - 私信
        
    - 留资
        
    - 把人引到企微
        

👉 **它是“流量入口”，不是成交系统**

---

## 三、你刚才纠结的那个销售逻辑，其实你判断是对的

你以为是：

> 秒红 → 秒回 → 成交

这个逻辑**未来是对的**，  
但**现在产品形态还没完全长到那一步**。

当前真实情况是：

- 市面上真正成熟在卖的：  
    **秒回 + 秒懂**
    
- 秒红：
    
    - 刚上线
        
    - 很初级
        
    - 偏试点
        
    - 还没形成标准销售链路
        

所以现在公司内部其实是：

> **先靠秒回活着，秒红在试水，秒懂是底座**

---

## 四、你可以用的对外标准说法

如果你要给别人讲，不会讲错版本是：

> 句子目前的核心体系是「秒懂 + 秒回」。  
> 秒懂负责定义 AI，秒回负责在企业微信里跑 AI。  
> 而秒红是后来为了抖音和小红书的公域获客场景，从秒回中拆出来的一条新产品线，用来把流量引进来，最终还是要回到秒回完成转化。

            │  - 角色                    │
            │  - 记忆                    │
            │  - 流程                    │

# 🧩 二、秒红在现在处于什么位置？

秒红不是这个体系的核心，它是一个**外挂流量模块**：

```
        抖音 / 小红书
              │
          （API / 私信）
              │
        ┌────────────┐
        │   句子秒红  │   ← 新产品，刚上线
        │ 公域获客层  │
        └───────┬────┘
                │ 传线索
                ▼
        ┌────────────────────────┐
        │        句子秒回           │
        │  私域成交 + 企微运营       │
        └────────────────────────┘
```

你刚才理解的那句话 **是 100% 对的**：

> **秒红就是：小红书 / 抖音版的秒回**

只是它：

- 不跑企微
    
- 不跑 RPA
    
- 只干一件事：  
    👉 **在公域把人“捞”出来**
