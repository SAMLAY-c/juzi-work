捎带案例的时候，感觉怪怪的；很明显看出来是ai写的
```
教育行业的话，我们产品在智能服务这块做得挺成熟的~ 比如说头部在线教育公司覆盖率已经达到80%了，能帮机构提升到课率、互动率、成单率这些核心指标，像到课率能提升60%，还有智能催到课、课后反馈这些功能，能减少人工成本，把人服比提高三倍左右，R值也能增加50%。对了，您公司叫啥名字呀？我备注下方便后续对接~
```

你非常敏锐，**机器人的工作是有边界的**。

SDR（销售开发代表）机器人的核心职责是**“清洗线索”**和**“确认意向”**（MQL 转 SQL），而不是**“完成成交”**（Close Deal）。

“已签约”、“流失”这些通常是销售介入后，经过谈判、报价、合同流程产生的结果，**机器人确实无法在前期判断**。

建议你把这个【SDR状态】字段拆解一下，或者制定一个**“状态流转规则”**。

---

### 方案一：拆分状态（最清晰，推荐）

不要把“机器人判断的状态”和“销售跟进的状态”混在一个字段里，逻辑会乱。建议把这个下拉菜单里的选项重新定义，**只保留机器人能判断的状态**，或者**按阶段划分责任人**。


| 状态名称 | 谁来操作 | 含义/触发场景 |
| :--- | :--- | :--- |
| **待清洗/新线索** | 系统默认 | 刚进来的线索，机器人还没聊。 |
| **跟进中 (AI)** | 机器人 | 机器人正在发消息，客户还没明确表态。 |
| **无效/垃圾** | 机器人 | **机器人判断**：发广告的、乱码的、完全无需求的。 |
| **有购买意向 (待人工)** | **机器人 (终点)** | **机器人判断**：客户留资了、要打电话、问价格了。👉 **这是机器人的工作终点，触发飞书推送。** |
| **跟进中 (人工)** | 销售 | **销售接手**：销售领走线索后，手动改为此状态，表示正在谈。 |
| **已签约** | 销售 | **销售判断**：合同签了，钱到了。 |
| **流失/关闭** | 销售 | **销售判断**：人工跟进后发现太贵不买、竞品截胡等。 |

---

### 方案二：自动化流转配置（SOP）

基于上面的状态定义，你的 Workflow 应该是这样的闭环：

1.  **机器人阶段（自动修改状态）**：
    *   当机器人判定为“广告” $\rightarrow$ 修改表格状态为 **【无效】**。
    *   当机器人成功索要到电话/判断高意向 $\rightarrow$ 修改表格状态为 **【流失】**。
    * 正常机器人和用户沟通--【跟进中】
2.  **人工阶段（手动/半自动修改状态）**：
    *   销售在飞书里点“我来跟进” $\rightarrow$ 自动把状态改为 **【已推送】**。






# 销售自己填
*   一个月后，销售谈成了 $\rightarrow$ 销售手动把状态改为 **【已签约】**。

---

### 总结建议

**不用删掉“已签约”，但要明确“分工边界”：**

1.  **机器人的任务**是把状态从 `空` 变成 `有购买意向` 或 `无效`。**它的工作到此结束。**
2.  **人的任务**是把状态从 `有购买意向` 变成 `已签约` 或 `流失`。

**给你的修改建议：**
在你的这个多维表格里，机器人只负责**写**前几个状态，后面几个状态留给销售去**改**。不要强求机器人在对话阶段就预知结果。
