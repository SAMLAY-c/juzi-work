# Obsidian中的组织方法 (Organizational Methods in Obsidian)

---

## **介绍 (Introduction)**

本笔记描述了可以在 Obsidian 中实施的主要组织方法，以改善知识管理、学习和项目管理。它们包括**卡片盒笔记法 (Zettelkasten)**、**康奈尔法 (Cornell Method)**、**帕累托原则 (80/20)**、**间隔重复 (Spaced Repetition)**、**看板 (Kanban)**、**深度提问 (Elaborative Interrogation)**、**番茄工作法 (Pomodoro Technique)** 和**费曼学习法 (Feynman Technique)**。在这里，我们描述如何将这些方法中的每一种应用到你的笔记和日常管理中。

This note describes the main organizational methods that can be implemented in Obsidian to improve knowledge management, studies, and projects. They include **Zettelkasten**, the **Cornell Method**, the **Pareto Principle (80/20)**, **Spaced Repetition**, **Kanban**, **Elaborative Interrogation**, the **Pomodoro Technique**, and the **Feynman Technique**. Here, we describe how to apply each of these methods to your notes and routine management.

---

## **1. 卡片盒笔记法 (Zettelkasten - 卡片方法)**

**概念 (Concept)**：每个笔记应该是一个"原子笔记"，包含一个想法。通过内部链接连接这些笔记，构建知识网络。

Each note should be an "atomic note," containing a single idea. Connecting these notes through internal links builds a network of knowledge.

### **如何在Obsidian中应用 (How to apply in Obsidian):**

- **原子笔记 (Atomic Notes)**：创建更小、更专注的笔记。每个笔记应该只处理一个概念。
- **内部链接 (Internal Links)**：通过 `[[Note_Name]]` 连接相关笔记，创建相互连接的想法网络。
- **MOC（内容地图）(Map of Content)**：为较大的主题创建 MOC 笔记，列出关于子主题的较小笔记的链接。
- **标签 (Tags)**：使用 [[标签和链接指南|Tags in Obsidian]] 对笔记进行分类

Create smaller, more focused notes. Each note should address only one concept.
Connect related notes through `[[Note_Name]]` to create a web of interconnected ideas.
Create MOC notes for larger topics, listing links to smaller notes about subtopics.
Classify notes with [[标签和链接指南|Tags in Obsidian]]

#### **示例 (Example):**
```markdown
# MOC - 网络安全 (Cybersecurity)
## 子主题：
- [[密码学]]
- [[防火墙]]
- [[DDoS攻击]]

## Subtopics:
- [[Cryptography]]
- [[Firewall]]
- [[DDoS Attacks]]
```

## **2. 康奈尔学习笔记法 (Cornell Method for Study Notes)**

**概念 (Concept)**：将笔记组织为三个部分：主笔记、关键要点和最终总结。

Organizes notes into three parts: main notes, key points, and a final summary.

### **如何在Obsidian中应用 (How to apply in Obsidian):**

- **主笔记 (Main Notes)**：放置讨论或学习的主题的地方。
- **关键要点 (Key Points)**：主要亮点或相关问题。
- **最终总结 (Final Summary)**：所学内容的结论或综合。

Where you place discussed or learned topics.
Main highlights or relevant questions.
Conclusion or synthesis of what was learned.

#### **示例结构 (Example Structure):**
```markdown
# [主题] 学习 (Study of [Topic])

## 主笔记：
- 概念 1
- 概念 2

## 关键要点：
- 见解 1
- 见解 2

## 总结：
- 我所学内容的总结。

## Main Notes:
- Concept 1
- Concept 2

## Key Points:
- Insight 1
- Insight 2

## Summary:
- Summary of what I learned.
```

## **3. 帕累托原则 (Pareto Principle - 80/20)**

**概念 (Concept)**：80%的结果来自20%的努力。通过识别最重要的笔记并重新组织它们以便轻松访问来应用这一原则。

80% of results come from 20% of efforts. Apply this by identifying the most important notes and reorganizing them for easy access.

### **如何在Obsidian中应用 (How to apply in Obsidian):**

- **笔记优先级 (Note Priority)**：使用 `#priority/high` 等标签标记最重要的笔记，并组织它们以便轻松访问。
- **重新组织 (Reorganization)**：创建"关键笔记"部分或使用 Obsidian 的收藏夹面板进行快速访问。

Mark the most important notes with tags like `#priority/high` and organize them for easy access.
Create a "Key Notes" section or use Obsidian's favorites panel for quick access.

#### **示例 (Example):**
```markdown
# MOC - 重要笔记
- [[信息安全]]
- [[C++优化]]

# MOC - Important Notes
- [[Information Security]]
- [[C++ Optimization]]
```

## **4. 间隔重复法 (Spaced Repetition Method)**

**概念 (Concept)**：以递增的间隔复习信息有助于长期记忆。

Reviewing information at increasing intervals helps with long-term retention.

### **如何在Obsidian中应用 (How to apply in Obsidian):**

- **间隔重复插件 (Spaced Repetition Plugin)**：使用 Obsidian 间隔重复插件设置自动复习提醒。
- **复习标签 (Review Tags)**：将 `#type/review` 标签添加到需要复习的笔记中。

Use the Obsidian Spaced Repetition plugin to set automatic review reminders.
Add the `#type/review` tag to notes that need reviewing.

#### **示例 (Example):**
```markdown
tags:
  - type/review
  - theme/algorithms
review_date: {{date}}
```

## **5. 看板 (Kanban - 任务板)**

**概念 (Concept)**：一种跟踪项目进度的可视化方法。非常适合划分任务和监控工作流程。

A visual method for tracking project progress. Ideal for dividing tasks and monitoring workflow.

### **如何在Obsidian中应用 (How to apply in Obsidian):**

- **Obsidian看板插件 (Obsidian Kanban Plugin)**：使用该插件创建看板并可视化任务。为项目阶段创建列，根据进度移动任务。
- **项目管理 (Project Management)**：非常适合管理游戏开发阶段或其他个人项目。

Use the plugin to create Kanban boards and visualize tasks. Create columns for project phases and move tasks according to progress.
Ideal for managing game development stages or other personal projects.

#### **示例 (Example):**
```markdown
# 看板 - 游戏开发 (Kanban - Game Development)
- **概念**:
  - [ ] 创建基本叙事
  - [ ] 定义核心机制
- **设计**:
  - [ ] UI原型
  - [ ] 角色建模

- **Concept**:
  - [ ] Create basic narrative
  - [ ] Define core mechanics
- **Design**:
  - [ ] UI prototype
  - [ ] Character modeling
```

## **6. 深度提问 (Elaborative Interrogation)**

**概念 (Concept)**：对内容提出深入的问题有助于理解和记忆。

Asking in-depth questions about the content helps with understanding and memorization.

### **如何在Obsidian中应用 (How to apply in Obsidian):**

- **笔记中的问题 (Questions in Notes)**：在学习笔记中添加问题部分，特别是关于复杂主题的。
- **反思 (Reflection)**：提出导致对内容进行更深入反思的问题。

Add a questions section in your study notes, especially about complex topics.
Ask questions that lead to deeper reflection on the content.

#### **示例 (Example):**
```markdown
## 问题：
- 如何将爱命运(Amor Fati)的概念应用到日常挑战中？
- 这如何影响我在压力下的决策？

## Questions:
- How can the concept of Amor Fati be applied to daily challenges?
- How does this influence my decisions under pressure?
```

## **7. 番茄工作法 (Pomodoro Technique)**

**概念 (Concept)**：将学习时间分为25分钟的专注块，随后是短暂的5分钟休息。在四个周期后，进行更长的休息。

Divide study time into 25-minute focus blocks, followed by a short 5-minute break. After four cycles, take a longer break.

#### **考虑超昼夜节律的变化 (Variation considering ultradian cycles)**

60-90分钟专注，15-20分钟休息（超昼夜节律）

60-90 min of focus for 15-20 min rest (Ultradian cycles)

### **如何在Obsidian中应用 (How to apply in Obsidian):**

- **任务列表 (Task List)**：为专注块创建任务列表并添加时间提醒。
- **计时器 (Timer)**：使用插件或外部应用程序控制番茄工作法周期时间。

Create a task list for the focus block and add time reminders.
Use a plugin or external application to control Pomodoro cycle times.

#### **示例 (Example):**
```markdown
# 番茄工作法任务 (Pomodoro Tasks)
- [ ] 学习数据结构 - 1个周期
- [ ] 复习C++概念 - 2个周期
- [ ] 研究网络安全 - 3个周期

- [ ] Study data structures - 1 cycle
- [ ] Review C++ concepts - 2 cycles
- [ ] Research network security - 3 cycles
```

## **8. 费曼学习法 (Feynman Technique)**

**概念 (Concept)**：把你正在学习的内容教给别人，就好像它很简单一样。这有助于识别理解中的空白并巩固知识。

Teach what you're learning to someone as if it were simple. This helps identify gaps in understanding and consolidate knowledge.

### **如何在Obsidian中应用 (How to apply in Obsidian):**

- **解释性笔记 (Explanatory Notes)**：撰写关于内容的解释，就好像你在教一个没有先验知识的人。
- **复习 (Review)**：始终审查你的解释以验证清晰度和知识空白。

Write explanations about the content as if you were teaching someone without prior knowledge.
Always review your explanations to verify clarity and knowledge gaps.

#### **示例 (Example):**
```markdown
# 解释网络安全 (Explaining Network Security)

网络安全就像保护房子，但不是窗户和门，而是防火墙和加密...

Network security is like protecting a house, but instead of windows and doors, you have firewalls and encryption...
```

## **结论 (Conclusion)**

这些方法是强大的组织工具，可以在 Obsidian 中一起应用，使你的"第二大脑"更加高效和动态。选择最适合你的日常工作的方法并根据需要进行调整。这些实践的结合将增强你的组织和信息保留能力。

These methods are powerful organizational tools that can be applied together in Obsidian, making your "second brain" more efficient and dynamic. Choose the methods that best adapt to your routine and adjust as needed. The combination of these practices will enhance your organization and information retention.
