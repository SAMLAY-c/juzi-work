# 一、这不是一张流程图，而是一套「AI 项目交付操作系统」

前一张图讲的是 **怎么跑**，  
这一张表讲的是 **每一步交付什么、给谁、怎么验收**。

两张图拼起来就是一句话：

> 句子的 AI 项目不是“上线一个功能”，

---

# 二、完整交付分三期（图 + 表的统一解释）

## 一期：前期 = 把 AI 员工造出来（能工作）

对应左侧大框「前期」。

|流程|做什么|产出什么|
|---|---|---|
|场景沟通|明确客户要干嘛|客户需求共识|
|资料收集|客户提供业务数据|**客户信息文档**|
|项目启动（1–3天）|方案 + 排期 + 测试 + 监控|项目启动方案|
|数据接口对接|接 CRM / 表单 / 聊天系统|接口联通|
|Demo 搭建|在秒懂里搭 AI|**功能实现记录文档**|
|Demo 测试|内测效果|**测试报告（对外）**|

👉 此时：  
你已经把「AI 员工 + 业务数据」拼起来了，但还不稳定。

---

## 二期：中期 = 用真实业务“训练 AI”（飞轮阶段）

这是最核心的价值创造区。

### 对应图中的飞轮

```
客户测试 → 问题反馈 → badcase → 调优 → 验收 → 再测试
```

### 对应表里的交付物

|模块|干什么|文档交付|
|---|---|---|
|验收|客户确认当前方案|**验收报告文档（对外）**|
|上线前验证|虚拟测试 / 灰度||
|业务规范|告诉客户怎么用|**业务验收以及上线须知（对外）**|
|问题记录|所有失败案例|**badcase 反馈表**|
|badcase 复盘|定期统计 + 客户会|**badcase 汇总报告**|

👉 这一阶段不是「修 Bug」，  
而是 **用客户真实数据驯化 AI**。

这是句子最值钱的部分。

---

## 三期：后期 = 固化 & 上线

当 badcase 飞轮跑顺，就进入“生产交付”。

|事项|含义|
|---|---|
|固定需求版本|这版 AI 能力锁定|
|邮件确认|双方对齐责任|
|上线投产|开始真实承接业务|
|每次更新记录|所有能力变化可追溯|
|功能实现记录文档|更新说明|
|文档记录|合规与交付依据|

👉 这意味着：  
你不是卖 SaaS，而是**交付一个可持续升级的 AI 员工版本**。
