## 前置准备：通用「提示词组件」

### 产品描述

想要让 AI 快速知道你的产品、场景、价值，最好提前准备一段固定的提示词，来介绍。

比如在句子互动，我们的句子秒回和句子秒懂两款产品的介绍是这样的。

### 通用场景及价值

同时，在解决方案中，不能只讲产品，也得将「用户场景」和『功能价值』。

所以，我们也得准备这两块的提示词组件。

```Markdown
# 产品通用场景：
被动触达 - 回应：根据工作要求、SOP、知识库，准确、及时、策略性地给予客户、领导、员工需要的内容。
主动触达 - 转化：通过设置各种触发器条件，我们的 AI 能够主动触达客户，并且通过策略性地对话、发送材料、发朋友圈等行为，引导客户从一种状态变成另一种状态（比如从萎靡到振奋，从不想购买到想购买等）。
内部预警：当某种特殊情况出现时，通过发消息、发邮件、打电话的方式，让需要了解情况的人快速知情。
仪表盘（BI）：将对话、产品和业务系统收集来的各种非结构化处理，变成结构化数据后进行计算，最后以 BI（数据仪表盘）的方式进行呈现，获得数据洞察。
汇报材料：汇报用的，利用 BI 看板和原始数据，用 AI 生成图文并茂的报告，可用于汇报、开会、上传下达，及时给客户、团队同步情况。
知识库：将过程中一切有利于未来业务的数据沉淀为知识资产（问答对、文档、表格、数据指标等），一方面给 AI 提供更多养料，一方面也能提高企业内员工的能力。
```

  

### 解决方案框架

我们的合作伙伴【有赞】内部有一套非常好的解决方案框架，用这套框架，结合客户的信息，可以非常好的归纳出解决方案的整体框架。

```Markdown
# 角色：
你是一个世界一流的 ToB SaaS 解决方案专家。

# 任务：
我将给你描述客户场景，请你根据我们的[解决方案框架]，用[风格示例]中的风格，帮我输出一份符合我们 SaaS 产品的场景解决方案。


#  解决方案框架：
我会告诉你用户场景，请你从以下几个维度，为我们输出场景解决方案。
1. 场景
含义：聚焦具体业务环节，提出运营中遇到的实际问题（通常以 “如何……？” 的疑问句呈现）。
写作逻辑：明确业务环节（如 “私域获客”“新客转化”“售后运营” 等）+ 具体问题（如 “多码混乱”“入会流程长”“导购售后无标准” 等）。
示例参考：“如何让客户一次扫码沉淀到多个私域触点？”（聚焦 “私域获客” 环节，问题是 “多码操作复杂导致沉淀难”）
2. 价值总结
含义：分为两部分 ——【能力】是解决方案赋予的核心功能 / 能力；【价值】是能力带来的业务收益（从效率、体验、转化等维度阐述）。
写作逻辑：先提炼「能力项」（动词 + 能力，如 “多码合一”“快速入会”），再对应「业务价值」（如 “提升客销力”“强化顾客信任”）。
示例参考：【能力】多码合一；【价值】客销力提升，高效连接顾客多个沟通端口（能力是 “多码合一”，价值是 “效率 + 客户连接” 的业务收益）
3. Before（优化前）
含义：描述解决方案引入前，场景中存在的痛点、问题、低效状态。
写作逻辑：用 bullet point 列举具体问题，包括流程繁琐点（如 “填多字段”“多后台操作”）、体验差细节（如 “操作不便”“等待久”）、业务损失（如 “转化率低”“数据混乱”）。
示例参考：
顾客扫三个码（企微码、公众号二维码、会员开卡码），操作不便，体验差
员工在三个后台看客流效果，麻烦（从 “用户操作”“员工效率” 两方面写实痛点）
4. Now（优化后）
含义：描述解决方案落地后，场景中流程、体验、效率的改进结果。
写作逻辑：用 bullet point 呈现改进动作（如 “一次扫码”“自动读信息”）+ 直接收益（如 “体验好”“沉淀 3 个触点”），突出 “解决方案如何解决 Before 的问题”。
示例参考：
消费者只需一次扫码，入会简便，体验好
一次获得消费者 3 个私域触点沉淀，便于后续运营（既写 “操作简化”，也写 “业务收益”）
5. 能力支撑
含义：实现 “Now” 状态所需的工具、系统或产品模块（通常是数字化工具 / 解决方案组合）。
写作逻辑：列举具体工具名称（如 “有赞 CRM”“企微助手”）或能力组合（如 “多码合一 + 企微活码”），体现技术对业务优化的支撑。
示例参考：有赞企微助手 + 多码合一 + 企微活码（明确工具 / 能力组合，说明 “技术如何支持业务改进”）
总结逻辑链
场景（提问题）→ 价值总结（给方向：能力 + 收益）→ Before（析痛点）→ Now（给方案：动作 + 结果）→ 能力支撑（给工具：技术保障）
这五个维度环环相扣，从 “问题” 到 “解决方案” 再到 “技术支撑”，完整呈现业务从 “痛点” 到 “优化” 的闭环逻辑。

# 风格示例：
这是我们和合作伙伴有赞写的产品解决方案，请你参考这个风格进行输出。
"
```

```Markdown
# 产品介绍：
句子互动分别有两款产品：句子秒回和句子秒懂。

## 句子秒回介绍：
句子秒回是一款 IM 聚合营销工具，主要功能为聚合多平台（包括飞书、钉钉、微信客服、微信公众号、微信小程序、抖音私信、小红书私信、WhatsApp、Tiktok 私信、Instagram 私信等聊天工具）和多平台私域营销自动化与智能化（包括批量私聊群发、群聊群发、关键词回复、自动踢人、智能回复、按照 SOP 发送私聊和群聊消息、利用大模型进行智能化私域影响等）功能。

句子秒回有以下特色功能：
1. 云端 SaaS 模式，无需下载客户端，支持随时通过浏览器使用，微信托管后硬件关闭不影响机器人运行；
自动化处理重复性工作（如自动加好友、打招呼、拉群、发送欢迎语、群满扩建等）；

## 句子秒懂介绍
句子秒懂是一个为企业打造的、具备主动服务能力和长期记忆系统的 Agentic 数字员工构建平台。
我们通过可视化的流程引擎、强大的 RAG（检索增强生成）能力、企业级的稳定性和独特的测试调优机制，帮助客户快速构建并落地真正懂业务、会思考、可持续优化的 AI Agent 应用。
核心亮点功能包括：

1. Agentic 流程引擎：不止于“编排”
- 主动触达与异步并发：Agent 不再被动等待，可基于事件（如用户询价后无响应）在特定时间 主动发起对话，推动业务目标达成。流程采用异步并发机制，大幅提升执行效率。
- 真人感 AI 对话：独创的智能消息合并 与分条发送功能，能将用户的多条信息整合理解，并模拟真人的思考和打字节奏进行回复，打造自然的“活人感”式聊天体验。
- 工具自主调用：Agent 能够自主决策，在对话中灵活调用知识库、数据表查询、网络搜索、代码执行等多种工具，更智能地完成复杂任务。

2. 知识库与记忆系统：让 Agent “活”起来
- 话术自学习：无需繁琐的手动录入，系统可直接从聊天记录中学习并自动提取、生成 FAQ 问答对，极大提升 Agent 的复刻销冠话术的能力。
- 智能查数 (Text-to-SQL)：突破传统 RAG 的文本检索限制，支持导入 Excel 等结构化数据。Agent 可理解用户问题并自动生成查询语句，精准回答基于数据的提问，从而完成复杂数据计算工作。
- 会话记忆系统：通过会话属性为每个用户建立专属的长期记忆。Agent 能记住用户的关键信息（如毕业院校、用户偏好），即使上下文滚动，也能提供连贯且个性化的服务。

3. 测试与调优中心：保证效果“不退转”
- 回归测试：业内独有的测试中心，可以将线上出现的 Bad Case 一键存为测试用例。每次优化后，系统会自动进行回归测试，确保 Agent 新增能力的在一直进步。

4. 客户洞察报告：从海量对话中获得局测洞察
- 定时分析，可自动对海量聊天记录进行语义分析、聚类和总结，定期生成客户洞察报告（如用户抱怨点、热门话题），并推送到指定渠道，为决策提供数据支持。

4. 企业级商用能力：稳定承载大规模业务
- 高并发处理：系统架构经大规模实战检验，可稳定承接数千人同时触发包含上百个节点的复杂流程，并支持百万级的批量任务执行，从容应对业务高峰。
- 插件与集成：灵活对接外部系统，查询或写入业务数据（如 CRM、ERP），将 AI Agent 无缝融入企业现有工作流，形成业务闭环。

# 产品通用场景：
被动触达 - 回应：根据工作要求、SOP、知识库，准确、及时、策略性地给予客户、领导、员工需要的内容。
主动触达 - 转化：通过设置各种触发器条件，我们的 AI 能够主动触达客户，并且通过策略性地对话、发送材料、发朋友圈等行为，引导客户从一种状态变成另一种状态（比如从萎靡到振奋，从不想购买到想购买等）。
内部预警：当某种特殊情况出现时，通过发消息、发邮件、打电话的方式，让需要了解情况的人快速知情。
仪表盘（BI）：将对话、产品和业务系统收集来的各种非结构化处理，变成结构化数据后进行计算，最后以 BI（数据仪表盘）的方式进行呈现，获得数据洞察。

知识库：将过程中一切有利于未来业务的数据沉淀为知识资产（问答对、文档、表格、数据指标等），一方面给 AI 提供更多养料，一方面也能提高企业内员工的能力。

#  解决方案框架：
我会告诉你用户场景，请你从以下几个维度，为我们输出场景解决方案。
1. 场景
含义：聚焦具体业务环节，提出运营中遇到的实际问题（通常以 “如何……？” 的疑问句呈现）。
写作逻辑：明确业务环节（如 “私域获客”“新客转化”“售后运营” 等）+ 具体问题（如 “多码混乱”“入会流程长”“导购售后无标准” 等）。
示例参考：“如何让客户一次扫码沉淀到多个私域触点？”（聚焦 “私域获客” 环节，问题是 “多码操作复杂导致沉淀难”）
2. 价值总结
含义：分为两部分 ——【能力】是解决方案赋予的核心功能 / 能力；【价值】是能力带来的业务收益（从效率、体验、转化等维度阐述）。
写作逻辑：先提炼「能力项」（动词 + 能力，如 “多码合一”“快速入会”），再对应「业务价值」（如 “提升客销力”“强化顾客信任”）。
示例参考：【能力】多码合一；【价值】客销力提升，高效连接顾客多个沟通端口（能力是 “多码合一”，价值是 “效率 + 客户连接” 的业务收益）
3. Before（优化前）
含义：描述解决方案引入前，场景中存在的痛点、问题、低效状态。
写作逻辑：用 bullet point 列举具体问题，包括流程繁琐点（如 “填多字段”“多后台操作”）、体验差细节（如 “操作不便”“等待久”）、业务损失（如 “转化率低”“数据混乱”）。
示例参考：
顾客扫三个码（企微码、公众号二维码、会员开卡码），操作不便，体验差
员工在三个后台看客流效果，麻烦（从 “用户操作”“员工效率” 两方面写实痛点）
4. Now（优化后）
含义：描述解决方案落地后，场景中流程、体验、效率的改进结果。
写作逻辑：用 bullet point 呈现改进动作（如 “一次扫码”“自动读信息”）+ 直接收益（如 “体验好”“沉淀 3 个触点”），突出 “解决方案如何解决 Before 的问题”。
示例参考：
消费者只需一次扫码，入会简便，体验好
一次获得消费者 3 个私域触点沉淀，便于后续运营（既写 “操作简化”，也写 “业务收益”）
5. 能力支撑
含义：实现 “Now” 状态所需的工具、系统或产品模块（通常是数字化工具 / 解决方案组合）。
写作逻辑：列举具体工具名称（如 “有赞 CRM”“企微助手”）或能力组合（如 “多码合一 + 企微活码”），体现技术对业务优化的支撑。
示例参考：有赞企微助手 + 多码合一 + 企微活码（明确工具 / 能力组合，说明 “技术如何支持业务改进”）
总结逻辑链
场景（提问题）→ 价值总结（给方向：能力 + 收益）→ Before（析痛点）→ Now（给方案：动作 + 结果）→ 能力支撑（给工具：技术保障）
这五个维度环环相扣，从 “问题” 到 “解决方案” 再到 “技术支撑”，完整呈现业务从 “痛点” 到 “优化” 的闭环逻辑。
```

5. ### 幻灯片故事线及排版设计思路
    

当解决方案思路较为确定时，可以让 LLM 出一套文字版的 PPT 排版思路，关键指令是👇🏻

```Markdown

# PPT 设计规范：
整体架构原则
采用金字塔原理构建整个PPT的叙事结构。从核心观点出发，逐层展开论证，确保每一页PPT都是故事线上的关键节点。页面之间必须形成紧密的因果关系、递进关系或并列支撑关系，让观众能够清晰感知到"为什么会有下一页"的逻辑必然性。整个演示应该像一部精心编排的电影，每个场景都为最终的高潮服务。

用户洞察要求
在PPT中大量融入真实的用户痛点、具体的使用场景和生动的用户故事。不要使用抽象的描述，而要用"张三在周一早晨遇到的困境"这样具象化的表达。每个痛点都要能让观众产生"这说的就是我"的共鸣感。用户故事要有细节、有情绪、有转折，让数据和观点变得有温度。

框架页设计标准
每个主要章节的开篇必须设置一页总揽性框架页。这一页要呈现该章节的完整逻辑架构，采用多层次、多维度的图表设计，可以是：

多层嵌套的模块图
复杂的流程架构图
立体的矩阵关系图
系统性的生态图谱
框架页要有苹果或特斯拉发布会级别的视觉冲击力，使用渐变、阴影、透明度等设计元素，让观众一眼就能感受到内容的专业性和复杂度。配色要大胆而克制，图表要精密而优雅。

内容页极简原则
除框架页外，每一页PPT必须遵循"一页一观点"的铁律。具体要求：

核心文字控制在15-25字以内
标题简洁有力，直击要害
如需辅助说明，使用高质量的图片、图标或数据可视化图表
留白面积不少于40%
视觉焦点唯一且明确
排版采用Bento风格设计理念：使用圆角矩形卡片、网格化布局、模块化分区，营造出整洁、现代、科技感的视觉效果。色块之间要有呼吸感，元素之间要有层次关系。

金句页设置规范
每个小章节（3-5页内容页为一个小章节）必须配置至少2页金句页，用于承上启下或强化记忆点。金句页设计要求：

文字字号超大（建议80pt以上）
金句长度控制在10-20字
文字要有态度、有力量、有记忆点
可以添加细微的装饰元素（线条、小图标、渐变背景）但不能抢夺文字的视觉主导地位
金句要么发人深省，要么激发情绪，要么总结精髓
金句的选择标准：能让观众拍照发朋友圈的句子。

整体质感标准
整个PPT要达到苹果发布会级别的质感，具体体现在：

配色方案：采用2-3个主色+黑白灰的配色体系，主色饱和度适中，整体色调统一
字体选择：使用现代无衬线字体，中文推荐苹方、思源黑体，英文推荐SF Pro、Helvetica Neue
动效暗示：在描述中标注适合使用渐显、推移等简洁动画的位置
图片质量：所有配图必须是高清、高质量的，风格统一
细节打磨：对齐、间距、比例都要精确到像素级
输出格式要求
请按以下格式输出PPT大纲：

第X章节：[章节名称]

设计元素：[具体的设计建议]
[页码] - 内容页：[页面标题]

核心观点：[一句话观点]
辅助元素：[图片/图表描述]
排版建议：[Bento风格的具体布局]

金句内容：「[金句文字]」
视觉氛围：[背景和装饰元素描述]
```
